# Deployment

Ross AI is designed for [Vercel](https://vercel.com) serverless functions.

## Deploy the app

1. Push the repo to GitHub
2. Import the project in Vercel (framework preset: **Next.js**)
3. Deploy — no build command changes needed

## Environment variables

In **Vercel → Project → Settings → Environment Variables**, add:

| Variable | Environments | Notes |
| --- | --- | --- |
| `OPENAI_API_KEY` | Production, Preview | Your OpenAI secret key |
| `NEXT_PUBLIC_APP_URL` | Production | e.g. `https://ross-ai-nine.vercel.app` |
| `NEXT_PUBLIC_DOCS_URL` | Production | e.g. `https://docs.ross-ai-nine.vercel.app` |

> If `OPENAI_API_KEY` is only set for **Preview**, production will return **503**. Enable it for **Production** and redeploy.

## Docs subdomain

To serve documentation at `docs.ross-ai-nine.vercel.app`:

1. Open **Vercel → Project → Settings → Domains**
2. Add domain: `docs.ross-ai-nine.vercel.app`
3. Assign it to the **same project** as the main app
4. Redeploy after the domain is verified

Middleware rewrites `docs.*` requests to `/docs/*` routes. The path `/docs` also works on the primary domain.

### Local subdomain testing (optional)

Add to your hosts file:

```
127.0.0.1 docs.localhost
```

Then run `npm run dev` and open `http://docs.localhost:3000`.

## Production limits

| Limit | Value | Why |
| --- | --- | --- |
| Max PDF size | 4 MB | Vercel serverless request body limit (~4.5 MB) |
| Contract text sent to OpenAI | ~15,000 chars | Cost control |
| Function timeout | 60 s | `maxDuration` in the API route |
| Rate limit | 10 requests / 15 min / IP | Abuse protection |

## PDF parsing on Vercel

Production uses **unpdf** (pure JavaScript) instead of `pdf-parse`, which relies on native bindings that fail in serverless Linux environments.

## Redeploy checklist

- [ ] `OPENAI_API_KEY` enabled for Production
- [ ] Latest commit deployed (includes unpdf fix)
- [ ] `docs.ross-ai-nine.vercel.app` domain added (optional)
- [ ] Test with a PDF under 4MB
